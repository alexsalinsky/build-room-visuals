LLM Builder Best Practices

A single file you drop into your AI build tool once. From then on, it documents every project you build, every session, automatically.

How to use it

  1. Download llm-builder-bps.md. One file. That's all you need.
  2. Drop it into your LLM chat (Claude, Codex, ChatGPT, any of them). Paste the file or upload it. Whichever your tool prefers.
  3. Tell it: Install this. Apply it to every session going forward. It will acknowledge once, then operate by these rules silently from that point on.

What's inside the file

Your AI gets a standing rule: maintain four living documents alongside every project it touches.

The four documents your AI will keep current

  • WORKBENCH.md Per-session log. What happened, what was decided, what's still open.
  • ARCHITECTURE.md The stack, the file map, the data flow, the key decisions.
  • USER_GUIDE.md Plain-language doc of what the thing does and how to use it.
  • TODOS.md An honest list of what's next. Next, Soon, Someday.

The file also installs honesty rules (no inventing file paths, no fake success claims), a clean communication style (lead with the answer, no preamble, no em dashes), and an automatic "wrap up" routine that fires when you end a session.

Why this works

  • Workbench. Captures the decisions and reasoning that git log doesn't.
  • Architecture. Stops you re-explaining your stack to your AI every Monday.
  • User Guide. Keeps you honest about what the thing actually does for the person using it.
  • TODOs. Stops you from carrying unfinished work in your head.

Together the four close the loop. Your AI builds, your AI documents, you trust the docs, the next session starts warm.
