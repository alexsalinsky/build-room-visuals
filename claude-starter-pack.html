Claude Starter Pack

One file. Save it, then add one line to your existing Claude or Codex config that points to it. Every session leaves behind a workbench entry, a current architecture doc, a user guide, and an honest TODO list.

What this is

One file, team-tech.md. All four document templates live inside it. You don't overwrite your existing CLAUDE.md. You point at this file with one line. Install takes 60 seconds.

The four documents Claude maintains

WORKBENCH.md
What just happened
Per-session log. 100-300 words. Newest at top. Decisions tagged with reasoning.
ARCHITECTURE.md
How the thing is built
Stack, file map, data flow, key decisions. Updated when the stack changes.
USER_GUIDE.md
What the user gets
Plain language. What it does, how to use it, settings. Updated on any user-facing change.
TODOS.md
What's next
Next / Soon / Someday. One line per item. No priorities, no labels, no dates.

Install (pick one)

Path A EVERY PROJECT (CLAUDE CODE)

  1. Download team-tech.md to ~/.claude/team-tech.md.
  2. Open ~/.claude/CLAUDE.md (create it if missing).
  3. Add this line anywhere in the file: @~/.claude/team-tech.md
  4. Done. Your existing CLAUDE.md rules stay; these layer on top.

Path B ONE PROJECT (CLAUDE CODE)

  1. Drop team-tech.md in your project's root folder.
  2. Open or create CLAUDE.md in the same folder.
  3. Add this line: @team-tech.md
  4. Done.

Codex users AGENTS.MD

Codex doesn't have a formal import directive. Two options:

  1. Paste: open team-tech.md, copy its contents, paste into your AGENTS.md under a ## Team Tech Rules heading.
  2. Reference: save team-tech.md next to AGENTS.md, then add a line at the top of AGENTS.md: Also follow the rules in team-tech.md (in this folder).

Most people start with Path B on one project, then graduate to Path A once the rhythm feels right.

Why each rule exists

  • Workbench. Captures the decisions and reasoning that git log doesn't.
  • Architecture. Stops you re-explaining your stack to Claude every Monday.
  • User Guide. Keeps you honest about what the thing actually does for the person using it.
  • TODOs. Stops you from carrying unfinished work in your head.

Together the four close the loop. Claude builds, Claude documents, you trust the docs, the next session starts warm.
