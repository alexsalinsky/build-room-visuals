We're learning that Cloudflare is a simple, fast route. If you already have a solution, great. This is for if you don't.

Four steps

It's very easy to sign up and get things running for free.

1
Sign up for a Cloudflare account Free tier. No credit card needed.
2
Create an API token One token, copied once, used everywhere.

Go to dash.cloudflare.com/profile/api-tokens → Create Token → pick the "Edit Cloudflare Workers" template.

FieldPick
Permissions Leave as the template gives them (Account: Workers Scripts Edit, Workers KV Storage Edit, Workers Tail Read, and the rest). Don't add or remove anything.
Account Resources Include → your Cloudflare account name (only one option).
Zone Resources Include → All zones. Covers you if you ever attach a custom domain. Doesn't matter for the free *.pages.dev URL.
Client IP Address Filtering Leave blank. Don't restrict by IP. You'll move between coffee shops, home, office.
TTL (Start / End date) Leave blank. Token never expires until you manually rotate it.

Click Continue to summary → Create Token. Copy the token shown. It's only displayed once.

3
Give the token to your robot Save it as CLOUDFLARE_API_TOKEN. Codex and Claude Code read it automatically.

Put it in your shell environment, your project's .env file, or wherever your robot looks for secrets.

Variable name: CLOUDFLARE_API_TOKEN. Value: the token you just copied.

4
Tell your robot to publish It uses the credential. You get a live URL.

Something like:

Publish this project to Cloudflare using my CLOUDFLARE_API_TOKEN. Give me the live URL.

Your robot handles the rest. The URL it returns ends in .pages.dev and is yours to share with anyone.

Bonus

Cloudflare has its own LLM service built in (Workers AI). Your robot can do AI things without hooking up additional API tokens for OpenAI or Anthropic.
