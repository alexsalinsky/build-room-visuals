Efficiency + ethicsAt publisher tier, two new costs show up. Name them. Plan for them.

So far you've been building for yourself. The thing runs when you press a button. The only person who sees the output is you. The cost is your own time.

At publisher tier, that changes:

New cost #1

Tokens

The thing runs without you watching it. Every run costs money. Scales with users.

New cost #2

Trust

Output reaches people who didn't agree to be in the loop. Whose data did it read? Who's accountable when it's wrong?

Efficiency: the three knobs

You don't need to optimize anything until you're at scale. But you DO need to know what dials exist, so you can reach for them when it matters.

Knob 1

Right tier model

Don't use the biggest model for a classifier. Sonnet for thinking, Haiku for sorting. Match the model to the job.

Knob 2

Right context

Don't ship your whole codebase as context every run. Chunk it. Cache it. Pass what's relevant, nothing else.

Knob 3

Right cadence

Does it really need to run every minute? Or once a day? Cron beats polling. Webhook beats cron when you can.

Ethics: the three questions BEFORE you ship

You don't need a 40-page policy document. You need to be able to answer these three questions, out loud, before you let anyone else use the thing.

Question 1

Whose data is in here?

Client emails? Payment records? Health info? Did the data owner know it'd be read by an LLM? If not, fix it before launch.

Question 2

Who can see the output?

Is there a path where the wrong person sees the wrong thing? A misrouted email, a public Slack channel, a logged-in account that gets shared.

Question 3

Who's accountable when it's wrong?

You? The user? The model? Say it plainly. "I built this. If it's wrong, it's on me." Or "User reviews before send. I'm not the last hop." Be explicit.

The pattern: one line at the top of your README

Add an accountability line to the README you wrote earlier. One sentence. Future-you and any teammate read it before they touch the thing.

This tool reads [WHAT data] for [WHO], produces [WHAT output] for [WHO ELSE]. [YOUR NAME] is accountable when it's wrong.

Concrete example:

This tool reads my own PayPal account, produces an internal CSV I review before any client is contacted. I'm accountable.

What you don't need to do yet

  • You don't need a privacy policy until external users sign up.
  • You don't need rate limiting until you have more than one user.
  • You don't need a refund flow until you charge.
  • You don't need SOC 2, ever, for a personal tool.

Build for the next user, not the thousandth one. But know which dials exist.

Efficiency keeps the thing affordable as it grows. Ethics keeps the thing trustworthy as it spreads. Both start the moment someone else uses it.
